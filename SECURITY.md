# 安全政策

## 支持范围

本仓提供通用 workflow/worker 运行时基础设施。安全修复优先覆盖默认分支和最新发布版本。

## 报告方式

请不要在公开 issue 中提交漏洞细节、真实凭据、token、cookie、验证码、数据库连接串、代理凭据或第三方平台会话材料。

推荐使用 GitHub Security Advisory 私下报告；如果仓库尚未开启该功能，请通过组织维护者提供的私有渠道联系。

## 处理原则

- 本仓计划开源，不引入私有契约仓、私有 SDK 仓或闭源业务仓。
- 日志、错误和测试样例不得包含可复用凭据或真实业务数据。
- 外部依赖必须有明确版本，并通过 Go module 管理。
